## 🎯 プロジェクトの目的

**店舗向けLINE抽選アンケートSaaS「ゲキッチャ」のMVPを作る**。

### ビジネスモデル
- 店舗がLINE公式アカウントを持っている前提
- 店舗は月額課金でゲキッチャを利用
- 店舗は友達に向けてアンケート＋抽選キャンペーンを打てる
- **最大の価値**: 匿名LINE友達を、当選のエサで「プロフィール入力済みの顧客DB」に変換できる

### ターゲット業種
飲食店、パチンコ屋、美容室、整体、リテールなど、LINE友達が数百〜数千いる個人店〜中小店舗。

## 🧠 重要な設計判断と背景

### 1. 演出のトーン
- 当初は「パチンコ風」演出だったが、ユーザーのフィードバックで **「応援・褒める」トーン** に変更
- 中盤は「応援ゲージ」で暖かい雰囲気
- 最後の抽選演出だけはパチンコ風に激アツ化（回答と抽選演出で緩急を作る）

### 2. 画像生成は Nano Banana Pro（Gemini 3 Pro Image）
- モデル名: `gemini-3-pro-image-preview`
- エンドポイント: `https://generativelanguage.googleapis.com/v1beta/models/gemini-3-pro-image-preview:generateContent`
- APIキー: **個人の `~/dev/.env.txt` の `GEMINI_API_KEY` に格納**（Gitには含まれない）
- 生成画像は **RGB のまま返ってくる（アルファチャネルなし）**
  - 「透過背景」と指示しても**灰色のチェック柄が描画される**
  - → `remove_bg.py` で灰色ピクセルを色差で検出してアルファ0に変換
  - 閾値: `abs(r-g) < 20 && abs(g-b) < 20 && 50 < r < 240`

### 3. レイアウトは CSS Grid + `clamp()`
- **全要素のフォントサイズに `clamp(min, preferred, max)`** を使ってどんな端末でも崩れない
- `.quiz` と `#complete` は **CSS Grid** で `grid-template-rows: 1fr ... 1fr` として上下の空白を伸縮させて中央寄せ
- **`.screen` は `position: absolute; inset: 0`** で全画面を占有。この上に `display: grid` などを重ねる
  - ⚠️ **`position: relative` を `.quiz` や `#complete` に追加すると画面が縮む**（過去バグ）

### 4. 文字は全て1行強制
- ユーザーの強い要望で **全てのテキスト要素に `white-space: nowrap`** を付与
- `<br>` は使わない

### 5. ロゴは画像にする
- 最初はCSS `text-shadow` で3D文字を表現していたが、クオリティ不足
- Nano Banana Pro で生成した画像に切替
- 結果: 「ゲキッチャ」「ありがとう」「抽選券」は全て画像

### 6. 公式LINEとゲキッチャの関係
- **店舗オーナーが公式LINEを所有**する構造（ゲキッチャ運営が乗っ取らない）
- ゲキッチャは「代理運用管理者」として招待してもらう建付け
- 既存の公式LINEとの共存前提（応答モードは「チャット＋Bot」）
- 詳細は explain.html のコンテ参照
